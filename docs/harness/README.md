## 文档结构

每个设计文档包含以下章节:

1. **问题分析**: Harness 理念 vs seed-agent 当前状态
2. **设计方案**: 核心类/函数设计、架构图
3. **实施步骤**: 分阶段任务清单
4. **预期收益**: 优化后获得的能力
5. **风险评估**: 潜在风险和缓解措施
6. **测试计划**: 单元测试/集成测试示例
7. **相关文档**: 依赖和关联文档链接

## 讨论要点

### 待确认问题

1. **Session 事件流**: 是否需要完整重放能力？
2. **Sandbox 隔离级别**: 进程级 vs Docker 容器级？
3. **L4 用户建模**: 辩证式进化的投入价值？
4. **凭证安全**: 对现有 API Key 流程的影响范围？
5. **实施优先级**: 资源分配和依赖调整？

### 已有优势 (无需优化)

| seed-agent 优势 | 描述 |
|-----------------|------|
| **RalphLoop 外部验证** | TEST_PASS/MARKER_FILE 等客观标准 |
| **SQLite+FTS5 中文支持** | jieba 分词，生产级存储 |
| **Memory Graph** | Laplace 平滑，历史成功率驱动 |
| **autodream ROI 评估** | 12 小时自动记忆整理 |
