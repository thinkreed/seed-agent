# 物理级全能进化型Agent 核心记忆原则与执行SOP
**适配路径**：`.seed/memory/`
**层级定义**：notes=L1 | skills=L2 | knowledge=L3 | raw=L4

---

## 一、核心记忆公理（最高优先级）
1. **行动验证原则**
   仅允许将**成功执行结果、探测数据、复盘结论**写入记忆，严禁存储臆测、常识、未执行计划。
2. **索引唯一入口**
   L1(notes) 为全局唯一索引，所有 L2/L3/L4 内容必须通过 L1 定位，无索引不调用。
3. **进化闭环原则**
   原始数据 → 操作SOP → 通用知识 单向沉淀，全程可追溯、信息不丢失。
4. **禁止存储易变/敏感信息**
   不存储临时PID、时间戳、绝对路径、密钥、会话ID等高频变化或敏感数据。
5. **高危操作标记**
   不可逆操作在 L1 索引中强制标注，执行前自动校验并确认。

---

## 二、层级定义与职责
| 层级 | 目录 | 定位 | 存储内容 |
|------|------|------|----------|
| L1 | notes | 全局索引层 | 极简关键词索引，指向 L2 技能、L3 知识、L4 原始数据ID |
| L2 | skills | 标准SOP层 | 可直接执行的操作步骤、技能流程、重试策略、失败处理方案 |
| L3 | knowledge | 通用知识库 | 从SOP提炼的规律、原理、环境配置、核心经验、通用规则 |
| L4 | raw | 原始归档层 | 数据库存储的原始会话、执行日志、探测原始数据、全量操作记录 |

---

## 三、层间关联规则
1. L1(notes) 只做**最小定位索引**，不存储任何细节内容。
2. L2(skills) 新增/修改SOP时，必须同步更新 L1 索引关键词。
3. L3(knowledge) 内容均来自 L2 高频复用SOP的提炼，L1 同步补充知识索引。
4. L2/SOP、L3/知识均可回溯至 L4 对应的原始数据ID，由 L1 统一关联。
5. 数据流转方向：**L4(raw) → L2(skills) → L3(knowledge)**，L1 全程索引同步。

---

## 四、Agent 记忆执行SOP
1. **任务启动**
   读取 L1(notes) 索引，通过关键词匹配 L2 技能 / L3 知识。
2. **执行过程**
   实时操作日志、原始数据写入 L4(raw) 数据库，临时推演不落地文件。
3. **任务完成**
   将有效流程整理为标准SOP存入 L2(skills)，并同步更新 L1 索引。
4. **自主进化**
   L2 中复用≥3次的SOP，提炼核心规律存入 L3(knowledge)，L1 补全知识索引。
5. **失败处理**
   失败原因与原始记录存入 L4，优化对应 L2 SOP，L1 标记异常关键词。
6. **任务闭环**
   必须完成 L1 索引更新 + L4 归档，未沉淀记忆则不结束任务。

---

## 五、强制约束（不可违反）
1. L1 仅做索引，禁止存放SOP、知识、原始日志。
2. L2 仅存可执行SOP，不存放纯理论知识。
3. L3 仅存提炼后的通用知识，不存放具体操作步骤。
4. L4 仅用于数据库归档原始数据，不直接参与执行调用。
5. 无 L1 索引关联的 L2/L3/L4 内容，视为无效记忆，禁止调用。
6. 所有记忆写入必须经过行动验证，禁止凭空生成。
