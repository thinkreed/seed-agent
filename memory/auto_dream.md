# 记忆整理 SOP (Auto Dream)
**路径**：`~/.seed/memory/` | **层级**：L1索引 | L2技能 | L3知识 | L4用户建模 | L5工作日志
**原则**：记忆只为**降错、提效、进化**服务，精简至上

---

## 层级职责

| 层级 | 职责 | 写入阈值 |
|------|------|----------|
| L1 | 存在性索引 | 删后犯错概率↑ |
| L2 | 可执行SOP | 复用≥3次 + 行动验证 |
| L3 | 通用规律 | L2复用提炼 |
| L4 | 用户建模 | 有偏好矛盾/强化证据 |
| L5 | 工作日志 | 有核心结论/关键发现 |

**数据流**：L5归档 → L4偏好洞察 → L2技能沉淀 → L3知识蒸馏 → L1索引同步

---

## ROI公式

**ROI = (犯错概率 × 操作代价) / 记忆体量成本**

---

## 四问校验法

1. 删掉本条，犯错概率↑？不上升 → 删
2. 已被L2/L3覆盖？已覆盖 → L1仅留触发词
3. 无此条，能自主想到查阅？能 → 删
4. 能用更少字表达？能 → 极致压缩

**L4/L5追加校验:**
- L4: 置信度>0.9已稳定？→ 标记固化
- L5: 90天未检索？→ 评估清理

---

## 写入必过

| 层级 | 规则 |
|------|------|
| L1 | 场景名触发词，无描述，≤30行(走次级索引) |
| L2 | 行动验证，可执行SOP，无理论 |
| L3 | L2复用≥3次提炼，无操作步骤 |
| L4 | 置信度≥0.6，矛盾触发升级(非覆盖)，例外上下文明确 |
| L5 | 自动归档，LLM摘要，90天无检索→候选清理 |

---

## 整理流程

1. **逐层标注**: L1→L2→L3→L4→L5 标类型
2. **低ROI清理**: L1-3剔除低价值，L4合并相似，L5清理90天+(保留100)
3. **补全高价值**: 补近期踩坑触发词
4. **合规检查**: L1≤30行 | L2=验证SOP | L3=规律 | L4=辩证偏好 | L5=归档摘要
5. **连通性**: L1→索引→L3/L5 链路完整

---

## L4辩证式规则

```
观察 → 矛盾检测 → 推理 → 升级(不覆盖)
```

- **无矛盾**: 强化置信度(+0.05)
- **有上下文矛盾**: 添加例外
- **置信度≥0.9 + 30天稳定**: 标记固化
- **置信度<0.5 + 90天无强化**: 候选清理

---

## L5归档规则

- **自动归档**: AgentLoop结束触发
- **LLM摘要**: 1-2句核心结论 + 3-5关键发现
- **FTS5索引**: jieba分词
- **清理**: 90天无检索 + 保留最近100

---

## 强制红线

1. 记忆修改=持久化，错误复利
2. L1禁操作细节，仅索引
3. 必须行动验证，禁凭空写入
4. 不可逆操作必标注风险
5. 禁敏感信息、易变状态
6. L4必须升级非覆盖
7. L5摘要禁手动删除
8. L4/L5操作必通过MemoryManager
