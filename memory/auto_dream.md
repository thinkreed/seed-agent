# 记忆整理 SOP (Auto Dream)
**适配路径**：`~/.seed/memory/` | **层级**：L1=索引 | L2=技能 | L3=知识 | L4=原始
**核心原则**：记忆只为**降错、提效、进化**服务，精简至上

---

## 一、层级定位与索引机制

### 1.1 层级职责
| 层级 | 职责 | 内容 |
|------|------|------|
| L1 | 存在性索引 | 只回答「什么场景下，有对应记忆可查」 |
| L2 | 可执行SOP | 解决「怎么做」—— 经行动验证的操作步骤 |
| L3 | 通用知识 | 解决「为什么」—— 跨任务通用规律、核心原理 |
| L4 | 原始溯源 | 仅复盘用，不参与执行 |

### 1.2 次级索引机制（L1 分层压缩）
L1 必须保持极简（≤30行），通过次级索引文件展开细节：
```
L1 (notes.md) → l1/l3-index.md → 具体 L3 文档
     ↓                ↓
  分类摘要        完整文件列表      知识详情
（名称+数量+关键词） （逐文件枚举）
```
- **L1** 只保留：分类名 + 文件数 + 关键词摘要 + 指向 `l3-index.md`
- **l3-index.md** 按分类存储完整 L3 文件名列表
- **L3 文档** 保持原位置，内容不变

---

## 二、ROI 评估模型

**ROI = (犯错概率 × 操作代价) / 记忆体量成本**

| 层级 | 核心标准 | 写入阈值 |
|------|----------|----------|
| L1 | 高容错/高价值，无此条会犯错 | 删后犯错概率↑ |
| L2 | 复用频次 ≥ 3 次 | 经行动验证的 SOP |
| L3 | 跨任务通用规律 | L2 复用提炼 |
| L4 | 溯源/复盘价值 | 关键失败记录 |

---

## 三、四问校验法（写入/保留前逐条校验）

1. 删掉本条，**犯错概率是否上升**？不上升 → 删
2. 是否已被 **L2/L3 覆盖**？已覆盖 → L1 仅留触发词
3. 无此条，能否**自主想到查阅 SOP**？能 → 删
4. 同等价值下，能否**用更少字表达**？能 → 极致压缩

---

## 四、各层写入规则

### L1 写入必过
- 能回答「这几个词对应**什么执行场景**」
- 触发词用**场景名**，不用工具/文件名
- 仅保留场景关键词，禁止描述性文字
- **红线规则**归入专项索引，SOP 仅保留 L2/L3 文件名

### L2 写入必过
- 必须经**行动验证**，无执行结果不写入
- 仅存可直接执行的 SOP，不存理论知识

### L3 写入必过
- L2 **复用≥3次** 的 SOP 提炼
- 只存通用规律，不存具体操作步骤

### L4 写入必过
- 仅存储有溯源/复盘价值的原始数据

---

## 五、标准化整理流程

1. **任务交付后**，按 L1→L2→L3→L4 逐条标注类型（红线/触发词/路由/细节/冗余）
2. **低ROI清理**：剔除低价值条目，校验「违反即炸还是仅好习惯」
3. **补全高价值**：结合近期失败/踩坑，补充遗漏的高ROI触发词
4. **层级合规**：L1≤30行(走次级索引) | L2=验证SOP | L3=通用规律 | L4=溯源数据
5. **连通性检查**：确保 L1→索引→L3 链路完整，无孤立条目

---

## 六、强制红线

1. 记忆修改为**持久化操作**，错误会复利影响后续执行
2. L1 严禁写入操作细节，仅做索引（走次级索引机制）
3. 所有记忆必须经**行动验证**，禁止凭空写入
4. 不可逆操作相关记忆，必须在 L1 标注风险标记
5. 禁止存储敏感信息、易变状态（临时路径/PID/时间戳）
