# 每小时代码优化 SOP

## 触发条件
- 定时任务 `hourly_optimization` 每小时触发

## 执行流程

### 1. 代码扫描
- 扫描 `src/` 目录下所有 Python 文件
- 排除 `core_principles/` 和 `golden_rules/` 目录（禁止修改）

### 2. 优化检查项

#### 代码质量
- [ ] 重复代码检测与提取
- [ ] 函数复杂度检查（超过50行考虑拆分）
- [ ] 命名规范（变量、函数、类名）
- [ ] 注释质量（删除冗余，补充必要）

#### 性能优化
- [ ] 循环优化（避免嵌套循环、提前计算）
- [ ] 数据结构选择（list vs set vs dict）
- [ ] 缓存机会识别
- [ ] 异步/并发优化机会

#### 错误处理
- [ ] 异常捕获完整性
- [ ] 边界条件处理
- [ ] 日志记录 adequacy

#### 类型与安全
- [ ] 类型注解完整性
- [ ] 输入验证
- [ ] 敏感信息暴露检查

### 3. 执行原则
- **全自动执行**：无需用户确认
- **小步提交**：每项优化独立提交
- **保守策略**：不确定的改动跳过
- **禁止修改**：`core_principles/` 和 `golden_rules/` 目录

### 4. 输出报告
执行完成后生成简要报告：
- 发现问题数
- 已优化项
- 跳过项（含原因）
- 建议后续关注点

## 注意事项
- 优先处理高影响、低风险的优化
- 保持向后兼容性
- 不破坏现有测试
